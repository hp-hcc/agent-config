# AGENTS.md instructions for C:\project\java-project

适用于 `C:\project\java-project` 下所有 Java 项目的通用 Java 代码约束。

## Java 代码约束

Java 代码需要满足以下约束（java-start-java-end）
java-start
新增或修改方法时，必须补充清晰的 JavaDoc 注释，写清方法职责、适用场景、`@param` 和 `@return` 的业务含义；必要时补充边界条件。
方法存在返回值时，JavaDoc 必须明确说明返回值含义；布尔返回值必须说明 `true` 和 `false` 分别代表什么。
修改已有代码时，如原有方法注释不清晰，且本次改动涉及该方法，应一并补全或修正注释。

方法参数为布尔值、状态值、范围值或其他魔法值时，必须在方法注释或调用处明确说明其业务含义。
调用方法时，应优先使用语义化变量，避免直接传入难以理解的字面量，例如 `xxx(false)`、`xxx(1)`、`xxx(2)`。
若确需直接传入字面量，必须在就近位置补充说明其业务含义。

业务逻辑方法注释规范：
适用于包含业务判断和分支逻辑的方法；不适用于简单 CRUD、纯查询透传、简单转换、getter/setter。
小型辅助方法注释应简洁，只描述当前方法判断什么或返回什么业务结果，不展开整条业务规则。
对关键业务分支，应在判断前增加就近中文注释，说明当前判断的业务原因和分支含义。
注释应使用业务语言，准确、简洁、直接，避免空泛描述、技术实现话术和重复代码表面意思。
同一类逻辑中的术语应保持一致，避免同义词混用。
java-end
